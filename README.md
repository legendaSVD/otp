# Erlang/OTP

Erlang/OTP - это исходники языка Erlang, runtime-системы и стандартных OTP-библиотек для построения отказоустойчивых и распределённых систем. В этой папке находится крупное дерево исходного кода с `erts`, стандартными библиотеками и build-инфраструктурой.

## ✨ Особенности

- Исходники runtime `erts`.
- Набор стандартных OTP-библиотек в `lib/`.
- Makefile-инфраструктура для множества подсистем.
- Код для компилятора, криптографии, common test, debugger и других компонентов.

## 🚀 Быстрый старт

### Что важно знать

Текущая копия выглядит неполной для полной сборки из корня: в корне есть `README.md`, каталоги `erts/`, `lib/`, `bootstrap/`, `bin/`, но отсутствует подтверждённый корневой `configure` или root-level manifest, с которого обычно стартует сборка всего репозитория.

### Что можно подтвердить по коду

- В `erts/` и `lib/` присутствуют многочисленные `Makefile`.
- В `erts/` присутствует локальный `configure`.
- Проект представляет собой исходное дерево Erlang/OTP, а не одно небольшое приложение.

### Запуск

Для этой конкретной копии папки корректный end-to-end запуск из корня подтвердить нельзя. Если у вас будет полный upstream-репозиторий, типовой сценарий обычно выглядит как сборка runtime и дальнейший запуск Erlang shell командой `erl`, но в текущем состоянии папки я не могу рекомендовать точную команду как гарантированно рабочую.

## 📁 Структура проекта

- `erts/` - runtime system и низкоуровневая инфраструктура Erlang.
- `lib/` - стандартные OTP-библиотеки.
- `bootstrap/` - bootstrap-материалы для сборки.
- `bin/` - вспомогательные скрипты и бинарные компоненты.
- `FILE-HEADERS/` - служебные шаблоны заголовков файлов.

## 🛠 Использование

Этот репозиторий используют как исходную базу для сборки Erlang/OTP и разработки внутренних библиотек/runtime-компонентов. После полноценной сборки основной пользовательский сценарий обычно связан с запуском `erl` и использованием модулей из `lib/`, но для этой локальной копии сначала нужно восстановить недостающие корневые build-файлы.
